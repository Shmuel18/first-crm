# פרומפט ביקורת לסוכן - מצב קריאה בלבד

החלף את `{מספר הסבב}` במספר המבוקש בין 1 ל-20.

```text
בצע את סבב מספר {מספר הסבב} בתהליך ביקורת ה-Production של first-crm.

הסבב מתבצע במצב READ-ONLY בלבד.

גבולות הרשאה מחייבים:

- אסור לשנות, ליצור, למחוק, להעביר, לשנות שם או לעצב מחדש שום קובץ.
- אסור לתקן קוד או מסמכים, גם כאשר התיקון ברור ופשוט.
- אסור לעדכן את docs/review/MASTER_LEDGER.md.
- אסור ליצור או לעדכן קובץ HANDOFF.
- אסור להוסיף, להסיר או לשנות בדיקות.
- אסור לבצע migrations, deploy, התקנות או שינויים בשירותים חיצוניים.
- אסור לשנות נתונים או מסד נתונים, ואסור לבצע בדיקות כתיבה מול Production.
- אסור לבצע פעולות Git שמשנות מצב: add, commit, checkout, branch, merge או push.
- אסור להריץ פקודה שעלולה לכתוב cache, artifacts, snapshots, נתונים או
  configuration ללא אישור מפורש לפקודה המדויקת.
- אם אינך בטוח האם פעולה עלולה לכתוב או לשנות דבר כלשהו, עצור ובקש אישור.
- אין לפרש את הבקשה לבצע ביקורת כאישור לבצע תיקונים.

קרא לפי הסדר:

1. AGENTS.md
2. CLAUDE.md
3. docs/REVIEW_PLAN_20_ROUNDS.md
4. docs/review/MASTER_LEDGER.md
5. docs/review/ROUND_HANDOFF_TEMPLATE.md
6. כל קובצי docs/review/ROUND-*-HANDOFF.md הקיימים
7. התיעוד הרלוונטי מתוך node_modules/next/dist/docs/ כאשר נבדקת התנהגות Next.js

לאחר הקריאה:

1. הרץ את פקודת הקריאה:
   node scripts/check-review-coverage.mjs
2. אשר מהו ה-Primary Scope של סבב {מספר הסבב}.
3. בדוק את git status בקריאה בלבד ושמור על כל השינויים הקיימים.

בצע שלוש מסירות:

מסירה א׳ - הבנה וביקורת:
- קרא כל קובץ שנמצא ב-Primary Scope של הסבב.
- מפה routes, actions, services, RPCs, migrations, הרשאות ותלויות.
- עקוב אחר זרימות חשובות מקצה לקצה.
- בדוק נכונות עסקית, אבטחה, פרטיות, אמינות, ביצועים, UX ותחזוקתיות.

מסירה ב׳ - אימות אדברסרי בטוח:
- נתח מקרי קצה, הרשאות חסרות, IDOR, דליפות מידע, race conditions,
  כשלים חלקיים, נתונים פגומים ובעיות אינטגרציה.
- בצע רק אימות סטטי או בדיקות שהן בוודאות קריאה בלבד.
- אל תבצע בדיקות הרסניות או בדיקות כתיבה מול Production.
- אמת ממצאים באמצעות ראיות. אל תציג השערה כממצא מוכח.

מסירה ג׳ - המלצות, תכנון רגרסיה והעברת מקל:
- אין לבצע תיקונים או שינויים בקבצים ללא אישור מפורש מהמשתמש.
- עבור כל ממצא מאומת, הצע תיקון ממוקד וציין אילו קבצים יושפעו.
- הצע בדיקות Regression שיוכיחו שהתיקון עובד.
- ציין סיכונים ותופעות לוואי אפשריות של התיקון.
- הכן תוכן HANDOFF מוצע לסבב הבא בתגובה בלבד.
- הכן שורות ועדכונים מוצעים ל-MASTER_LEDGER בתגובה בלבד.
- עצור והמתן לאישור לפני כתיבה או שינוי כלשהם.

לכל ממצא כלול:

- מזהה
- חומרה: Blocker / Critical / High / Medium / Low
- סטטוס: Suspected או Verified
- רמת ודאות
- קובץ ומספר שורה
- ראיה או דרך שחזור בטוחה
- השפעה עסקית או טכנית
- תיקון מוצע
- בדיקת Regression מוצעת
- Owner Round אחד
- Verification Round נוסף עבור Critical או High
- חוזים ותלויות בין סבבים

בסיום הצג בתגובה בלבד:

1. היקף וקבצים שנבדקו.
2. אזורים שלא נבדקו והסיבה.
3. ממצאים לפי סדר חומרה.
4. שורות ועדכונים מוצעים ל-MASTER_LEDGER.
5. תוכן HANDOFF מלא ומוצע בהתאם לתבנית.
6. פקודות קריאה בלבד שהורצו ותוצאותיהן.
7. פקודות או בדיקות שלא הורצו משום שהן דורשות אישור.
8. האם הסבב הבא יכול להתחיל בבטחה.

אל תשנה דבר. לאחר הצגת הדוח, עצור והמתן לאישור מפורש וספציפי.
רק סבב 20 רשאי להציע החלטת GO / CONDITIONAL GO / NO-GO.
```

```text
- [רשימת קבצים מדויקת]

אסור לבצע שינויים או פקודות נוספות ללא אישור חדש.
```
