## F1 — תצוגת שמות לווים: "+1" בדשבורד, שני שמות בכותרות דפים ✅

- **סטטוס:** ✅ **בוצע + תוקן** (אחרי משוב) — פוצל לשתי פונקציות:
  - **דשבורד** (טבלה צפופה) — `getCaseClientLabel` = "+1" כמו קודם (חזר אחרי
    שהמשתמש ביקש; שינוי F1 המקורי בטעות הפך גם אותו לשני שמות).
  - **כותרות דפי הסימולטור** — `getCaseClientLabelFull` = עד 2 שמות מלאים (" · "),
    "+N" מ-3 ומעלה. אומת בדפדפן (דשבורד "+1" · כותרת מיקס "כהן יוסי · כהן שרה").
  - כותרת התיק הראשית + דף המסמכים כבר מציגות את כל השמות עם " & " (לא הושפעו).
    אם רוצים גם אותן בפורמט "2 שמות +N" — שינוי קטן, פתוח.
- **קבצים:** `case-derivations.ts` (+ בדיקות), 5 דפי `simulators/*/page.tsx`.
- **בעיה:** בכותרת התיק (סרגל הפעולות, וגם בדשבורד) מוצג שם הלווה הראשי + "‎+1".
  היועץ לא יודע מי הלווה השני בלי להיכנס.
- **החלטה:**
  - לווה 1 → שם מלא (ללא שינוי)
  - 2 לווים → שני השמות המלאים, מופרדים ב-" · "
  - 3+ → שני השמות הראשונים + "‎+N" של היתר
  - הסף (2) ניתן לכוונון; ברירת מחדל = 2 שמות מלאים.
- **איפה בקוד:** `src/features/cases/domain/case-derivations.ts` →
  `getCaseClientLabel` (שורות 39–58). פונקציה מרכזית אחת — מתקנת גם את
  סרגל הפעולות (`case-action-bar.tsx`) וגם את הדשבורד. לעדכן את הבדיקות
  ב-`case-derivations.test.ts`.
- **אומדן:** קטן (פונקציה + בדיקות, ללא מיגרציה, ללא UI חדש).
- **הערה:** הסרגל כבר עוטף ב-`truncate max-w-md`, אז שני שמות ארוכים לא ישברו.

  - שליש = ‎33.3333…%‎. הבדיקה ([regulatory-rules.ts:52]) היא `actual < limit`
    **בלי טולרנס** → ‎33.3333… < 33.3334‎ = TRUE → חריגה שגויה.
  - אותו סוג כשל אורב גם בגבול העליון (פריים 2/3 = ‎66.6667‎): ערך גבול מדויק
    עלול ליפול בצד הלא נכון של `>`/`<`.
- **תיקון מוצע:**
  - **עיקרי:** טולרנס (epsilon) בהשוואות ב-`shareViolation` — למשל
    `actual < limit - 1e-6` / `actual > limit + 1e-6`, או השוואה על סכומים
    (`fixedAmount * 3 >= total`) כך ש"בדיוק שליש/שני שליש" תמיד עובר.
  - **משני:** לתקן את ערך הסף בזרע (33.3334 → 33.3333 או נמוך), וליישר בין
    ברירת המחדל בקוד לערך ב-DB. דורש מיגרציה + החלה על DEV+פרוד.
- **איפה בקוד:** `src/features/simulators/domain/regulatory-rules.ts`
  (`shareViolation`), `constants.ts` (`DEFAULT_REGULATORY_THRESHOLDS`),
  מיגרציה 094 (הערך ב-DB). יש קובץ בדיקות `regulatory-rules.test.ts` —
  להוסיף מקרה-גבול של בדיוק שליש/שני-שליש.
- **אומדן:** קטן (טולרנס בקוד + בדיקות). תיקון ערך ה-DB = מיגרציה קטנה נוספת.
